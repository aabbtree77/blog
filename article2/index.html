TypeScript is fine for a frontend, but what to choose for a backend?! A few considerations:

TypeScript

  • Example: SaaS with OpenAI Chat Bot.

  • Js, React, and Next.js fatigue.

  • Scary serverless DDOS stories.

  • Minimal RAM: 100MB.

  • Proponents: Josh tried coding, Ben Davis, Your Average Tech Bro, Marc Lou, syntax.fm…

Go

  • Example: PocketBase.

  • Typically, a minimalistic router with templ, htmx, ORM or no ORM. SSR (MPA)-mostly. YAGNI, KISS, DIY.

  • Go is not perfect. F#, Rust, Gleam may have superior type systems and fewer glitches, but I would choose Go for not being too academic or overengineered.

  • Minimal RAM: 10MB. A single binary!

  • Proponents: Web Dev Cody, ThePrimeagen, Anthony GG, Adrian Hesketh…
